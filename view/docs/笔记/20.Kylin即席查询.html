# Kylin简介

开源分布式分析引擎，提供hadoop/spark之上的SQL查询接口及多维分析(OLAP)的能力。

亚秒(比秒更快一点)级内查询巨大的HIVE表

数仓是BI的核心组件

BI:商业智能,将数据转换成知识。为企业做决策的工具

涉及数仓、联机分析处理(0LAP)、和数据挖掘等技术。

OLAP:是种软件技术，可让分析人员从各个方面观察信息。故称多维分析。

  • 类型

    ROLAP:基于关系型数据库，不需要预计算。

    MOLAP:基于多维数据集，需要预计算。

    一个多维数据集，称为OLAP Cube

    由多个cuboid组成一个OLAP Cube

kylin对接的就是维度模型

维度：分析数据的角度

度量：被分析的指标

# 架构

  • 查询层=>路由层=>计算层

特点

  • 提供标准的SQL的接口

  • 支持超大数据集

  • 亚秒级响应

  • 可伸缩性和高吞吐率

  • BI工具集成

    例:

    ODBC: Tableau、excel、powerBi等

    JDBC：Saiku、BIRT等java工具

    RestAPI：javaScript、web网页等

# Cube构建原理

维度和度量：观察角度、计算结果

cube和cuboid：每一种维度组合就是一个Cuboid，16个Cuboid整体就是一个Cube。

构建算法：

  • 逐层构建layer

    • 优点：充分利用MR优点，处理了中间复杂的排序和shuffle工作。慢、稳定
    • 缺点，当维度较多，mr任务增加需要额外计算资源，未聚合，每轮shuffle工作量大，对hdfs读写多

    该算法的效率较低，尤其是当Cube维度数较大的时候。

  • 快速构建算法 inmem

    • 会利用内存做预聚合，算出所有组合，mr输出的key都不同，见扫mr数量，不需要combiner，快，不稳定

可以让它自动选择，也可以指定

存储原理

创建维度字典表，真正的值就不会参与运算。

存储Hbase K-V

Cuboid+纬度值

# 优化手段

优化计算or查询

聚合组：剪枝工具 计算

减掉没有意义的维度组合

  • 强制维度：使用不同的强制维度(选择跟该维度依赖相关的值) A
  • 层级维度：跟层级有关，前后关系，例：时间、地区、部门 ：A=>B
  • 联合维度：将维度看作整体,例：AB

配置在构建cube的第五步

衍生度： 减少Cuboid的数量

  • 衍生维度不会参与最终的计算当中。

  • 事实表与该维度表中关联的ID外建参与计算

  • 但查询需将将对应的ID转成具体的维度可能会做个聚合操作，所以，从维度表主键到某个维度表所需的聚合工作量大，不建议使用衍生度

一般情况可以，不用设置，如果任务量太大才需要设置，因为查的慢

Row key 优化：

  • 被用作过滤的维度放在前边。 ：优化查询
  • 基数大的维度放在基数小的维度前边。：优化计算

在页面上下拖动调节即可
