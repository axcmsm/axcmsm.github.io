# 简介

HBase 是一种分布式、可扩展、支持海量数据存储Nosql数据库

原谷歌三大论文之一Bigtable演化成HBase (GFS是hdfs,mr就是mr)

基于hdfs,可以实现随机写操作。秒级响应(方式合久必分，分久必合)

数据结构：逻辑结构表，物理存储结构KV

逻辑结构：元数据

将很多列分出来组成一个列族,从上往下叫Row key 按照字典序排序，而且唯一,不唯一就是覆盖。

(宽表)纵向切分叫列族,(长表)横向切分的叫Region，切分后都放置不同文件夹。实际数据存储到Store里面

存储结构：

对于每一个列:都存储了列族、列名、Rowkey、timeStamp时间戳(版本号)、操作类型、值

最后会校验时间戳(返回时间最大的)，如果不一致操作会出问题

删除操作:rowkey相同,类型是delet,然后比对时间戳(如果是最大)执行成功,返回删除

不会立马删除

数据模式：

  • Name Space :命名空间 类似database

    默认：hbase(内置表) 、default(默认用户使用的命名空间)

  • Region：表 的切片(小表) 、定义列族即可、列是动态增加的

  • Row：每行数据又一个rowkey(字典序)和多个列组成

  • column:每个列需要指定列族与列限定符

  • Time stamp:时间戳标识版本

  • cell:唯一确认的单元(最小)

基础架构：

master操作表的DDL，zookeeper管理,其中storeFile存储到Store，Store存储到Region,Region存储到RegionServer 分布式，

RegionServer作用:DML

  • Data:get、put、delete
  • Region：SplitRegion、compactRegin

master作用：DDL

  • table:create、delete、alter
  • RegionServer：分配Region到每个RegionServer,监控RegionServer状态

自带高可用(俩个master)

最终文件存储到hdfs,而hdfs存储到datanode中，而datanode存储到linux中,然后存储到磁盘上。

# 进阶

架构原理:

数据最后存储到hdfs的datanode中,HMaster DDL管理表,管理HRegionServer,ZK分担工作量，HRegionServer包含Hlog(预写入日志)、多个HRegin(按行切分,里面包含多个列族sotre.memstore(刷写),stroeFilme(文件))里面的存储格式是Hfile(键值对),列族是逻辑结构真实文件夹是store

zk分担了HMaster与客户端的交互，l客户端直接跟zk交互，

如果操作是DML找zk,DDL还是需要跟HMaster交互

memstore刷鞋过程要经过跟hdfs客户端交互最后刷写到datanode

也就是磁盘上。

写流程：RegionServer简称RS

客户端向zk请求meta(结构)表所在的的RS(谁在维护),假设zk返回hadoop111(它在维护)，客户端再向Hbase中请求meta(表信息),hbase返回meta,获取RS(数据所在位置),做meta缓存，再发送put请求,先写入预写日志,再写入memstore缓存,再写入storefile文件。

在0.9之前还有-root-表 其他表往后推一步。

现在改变了，在切分之前会先判断是否是meta表

刷写流程：

RS级别：里面有多个hr

默认占超过RS的堆内存的40%刷写到磁盘(内存堵塞行为)

安全模式,占堆内存的95%(也就是0.38)也开始刷写(没有堵塞直接刷写,一边写入缓存，一边刷写磁盘,当写入缓存速度比写磁盘快,到堆内存40%，进入堵塞,不要往内存写,不然爆了,等到0.38以下再开启写入缓存)

当内存最后一次操作时间,过了一个小时(认为没有操作,也会进行刷写)

hr级别：hr中的memstore

当前的memstore到达了128M,单个进行刷写(里面还有wal预刷写操作)

读流程：

先读内存，再找zk,找meta位置,请求meta查找RS位置,缓存位置,再请求对应RS,其中RS中有一块Block Cache用来缓存最近读取的数据(遵循最近最少原则,满内存就开删一部分),读取这个缓存，也去读磁盘数据,然后进行比对时间戳。

数据量大的几乎都是读磁盘的,所以hbase是一个读比写慢的框架

合并数据：storeFile Compaction

  • Minor Compaction 小合并(不删数据)

  • Major Compaction 全局合并(合并成一个) compact '表名'

    生产情况建议设置为0,关闭自动合并,以为是重写,先读再写,影响业务，建议在空闲时间,手动全局合并

    全局合并后应该需要等几分钟才会自动删除源文件

切片流程：Region Split (默认10G)

0.94之前到达之后才且,0.94后，自动按照缓存最大值切分为2,一直下去，会操作数据倾斜问题,所以在建表时就要做预分区。

到达值,按照整个rowkey一刀俩断

官方建议，经量使用一个列族(使用多个尽量保证数据同步增长,防止小文件)

package com.axcmsm.test;
import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.hbase.*;
import org.apache.hadoop.hbase.client.*;
import org.apache.hadoop.hbase.util.Bytes;
import java.io.IOException;
import java.util.Iterator;

/*
 * DDL：
 * 1判断表是否存在
 * 1.创建表
 * 3.创建命名空间
 * 4.删除表
 *
 * DML:
 * 1.插入数据
 * 2.查数据(get)
 * 3.查数据(scan)
 * 4.删除数据
 *
 *
 * */
public class TestApi {
    private static Connection connection = null;
    private static Admin admin = null;

    static {
        //        HBaseConfiguration conf = new HBaseConfiguration();
        Configuration conf = HBaseConfiguration.create();
        //连接zk
        conf.set("hbase.zookeeper.quorum", "hadoop111,hadoop112,hadoop113");
//        HBaseAdmin admin = new HBaseAdmin(conf);
        try {
//           创建连接对象
            connection = ConnectionFactory.createConnection(conf);
        } catch (IOException e) {
            e.printStackTrace();
        }
        try {
//            创建admin对象
            admin = connection.getAdmin();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }


    //    1判断表是否存在
    public static boolean isTablieExit(String tableName) throws IOException {
        boolean flag = admin.tableExists(TableName.valueOf(tableName));
        return flag;
    }

    //关闭方法
    public static void close() {
        if (admin != null) {
            try {
                admin.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
        if (connection != null) {
            try {
                connection.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }

//    创建表
    public static void createTable(String tableName,String... cfs) throws IOException {
        if (cfs.length<=0){
            System.out.println("请设置列族信息");
            return;
        }
        if (isTablieExit(tableName)){
            System.out.println("表已经存在");
            return;
        }
        HTableDescriptor table = new HTableDescriptor(TableName.valueOf(tableName));
        //循环添加列族
        for (String cf : cfs) {
            //创建列族描述器
            HColumnDescriptor family = new HColumnDescriptor(cf);
            table.addFamily(family);//添加列族信息
        }
        admin.createTable(table);
    }

    //删除表
    public static void dropTable(String tableName) throws IOException {
        if (!isTablieExit(tableName)) {
            System.out.println("不存在");
            return;
        }
        //下线
        admin.disableTable(TableName.valueOf(tableName));
        //删除表
        admin.deleteTable(TableName.valueOf(tableName));

    }
    //创建命名空间
    public static void createNameSpace(String ns){
        try {
            admin.createNamespace(NamespaceDescriptor.create(ns).build());
        }catch (NamespaceExistException e) {
            System.out.println(e+"命名空间已存在！！！！！！！！！！");
        }
        catch (IOException e) {
            e.printStackTrace();
        }
    }

    //插入数据
    public static void putData(String tableName,String rowKey,String cf,String cn,String value) throws IOException {
        Table table = connection.getTable(TableName.valueOf(tableName));
        Put put = new Put(Bytes.toBytes(rowKey));//传入rowkey
        put.addColumn(Bytes.toBytes(cf), Bytes.toBytes(cn), Bytes.toBytes(value));
        table.put(put);//插入数据
        //关闭表连接
        table.close();
    }

    //获取数据
    public static void getData(String tableName,String rowKey,String cf,String cn) throws IOException {
        Table table = connection.getTable(TableName.valueOf(tableName));
        //指定rowkey
        Get get = new Get(Bytes.toBytes(rowKey));
        //        指定列族  (不指定默认按row获取)
//        get.addFamily(Bytes.toBytes(cf));
        //指定列族和列
        get.addColumn(Bytes.toBytes(cf),Bytes.toBytes(cn));
        //获取数据的半本数
        get.setMaxVersions(5);//最大半本数
        //  解析
        Result result = table.get(get);
        for (Cell cell : result.rawCells()) {
            System.out.println("CF:" + Bytes.toString(CellUtil.cloneFamily(cell)) +
                    ", CN: " + Bytes.toString(CellUtil.cloneQualifier(cell))+
                    ", value: "+Bytes.toString(CellUtil.cloneValue(cell)));
        }
        table.close();
    }

    //获取数据
    public static void scanData(String tableName) throws IOException {
        Table table = connection.getTable(TableName.valueOf(tableName));

        //空参全表  可以传入rowkey 和列族 还有过滤器
        ResultScanner scanner = table.getScanner(new Scan(Bytes.toBytes("1001")));
        for (Result result : scanner) {
            //解析result
            for (Cell cell : result.rawCells()) {
                System.out.println("Rk:"+Bytes.toString(CellUtil.cloneRow(cell))+
                        ",CF:" + Bytes.toString(CellUtil.cloneFamily(cell)) +
                        ", CN: " + Bytes.toString(CellUtil.cloneQualifier(cell))+
                        ", value: "+Bytes.toString(CellUtil.cloneValue(cell)));
            }
        }
        //关闭连接
        table.close();
    }

    //删除数据
    public static void deleteDate(String tableName,String rowkey,String cf,String cn) throws IOException {
        Table table = connection.getTable(TableName.valueOf(tableName));

        // 只传入rk 就相当于deleteall
        Delete delete = new Delete(Bytes.toBytes(rowkey));
        //指定列族、列删除新版本   慎用 你删除新版本旧版本又出来了
//        delete.addColumn(Bytes.toBytes(cf),Bytes.toBytes(cn));
        //指定列族、列删除全部版本
//        delete.addColumns(Bytes.toBytes(cf),Bytes.toBytes(cn));
        //指定列族
        delete.addFamily(Bytes.toBytes(cf));
        table.delete(delete);
        table.close();

    }

    public static void main(String[] args) throws IOException {
//        System.out.println("------------------");
////        System.out.println(isTablieExit("0408:stu5"));
//        System.out.println("------------------");
        //创建表测试
//        createTable("0408:stu5","info1","info2");
//        createTable("0408:stu5","info1");
        //删除表
//        dropTable("0408:stu5");
//        //创建命名空间
////        createNameSpace("0408");

////        插入数据测试
//        putData("0408:stu5","1001","info1","name","lisi");
//        putData("0408:stu5","1001","info1","age","12");
//        //获取单行数据
//        getData("0408:stu5","1001","info1","name");
        //获取全表数据
        scanData("0408:stu5");
        System.out.println("------------------");
//        System.out.println(isTablieExit("0408:stu5"));
//        System.out.println("------------------");

    }
}

# 整合Hive

hbase和hive比较：

habase是用来存储数据的,面向列族存储的非关系型数据库,存储结构化和非结构化的数据,适用于单表查询,基于hdfs,延迟低,接入在线业务适用(类似 es、redis)

Hive是分析清洗数据的,做数据仓库。
