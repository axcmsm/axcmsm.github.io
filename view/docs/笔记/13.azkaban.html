# azkaban

官网https://azkaban.github.io/ (opens new window)

用作调度hadoop的工作流程

常见工作流调度系统：Crontab(简单任务)、Ooize、azkaban、airflow、dolphinscheduler等

oozie：oozie使用xml配置,在cdh中也可以使用hue组件进行可视化操作
azkaban：yaml配置
airflow：python脚本
dolphinscheduler:可视化

# 条件工作流

按照自定义条件执行job任务

条件构成：

  • (多个)父任务运行结束输出的参数
  • 使用预定义宏(使用它提前预定的内置规则)

基本原理:

（1）父 Job 将参数写入 JOB_OUTPUT_PROP_FILE 环境变量所指向的文件 json字符串 
（2）子 Job 使用 ${jobName:param}来获取父 Job 输出的参数并定义执行条件

再使用条件运算符进行必对执行.
1
2
3
4

步骤：

  • 编写父任务脚本 json字符串k,v > $JOB_OUTPUT_PROP_FILE

  • 子任务脚本

  • 配置文件

    #子任务添加
condition: ${父任务名称:在文件中的key} == 值   
 #这里是去JOB_OUTPUT_PROP_FILE根号key拿到value 在进行条件判断
    1
    2
    3

  • 打成一个zip 上传即可

预定义宏：根据所有父job的完成情况进行判断

参数：

1.all_success: 标识父job全部成功才可以
2.all_done: 表示父job全部完成才执行
3.all_failed: 表示父job全部失败才执行
4.one_success: 表示job至少一个成功才执行
5.one_failed: 表示父job至少一个失败才执行

# 注意事项

在集群模式下,多台exec执行任务时,是webserver选择其中一个exec去执行。(大概率执行失败)

因为你执行的脚本不在同一个exec节点下。

解决方案：

  • 指定特定exec去执行 （id不固定,负载均衡失效...）

    在mysql的`azkaban数据库的executors表中,查询hadoop111的exec的id
在web页面执行工作流程时添加useExecutor的属性 把id放里面
#id每次启动又概率会发送改变

    2
    3

  • 所有的exec节点都添加该脚本 (推荐)
