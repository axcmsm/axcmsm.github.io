# Presto简介

是开源分布式SQL查询引擎，数据量GB到PB字节之间，主要处理秒级查询的场景。

充分利用内存，没有用到预计算

注意：它只负责计算查询，不负责存储，不支持在线事务()。

# 架构

优点：

  • Presto基于内存运算，减少IO,计算更快
  • 能够对接多种数据源，跨数据源联表查询，例：hive和mysql连表

缺点：

  • 不是一下子全部放入内存，是边度数据边计算，再清内存，再读再计算。
  • 当链表查，可能参数大量临时表，查询会变慢。

可以在hive先把需要查询的表join到一起形成大宽表，迎合其特点。

与impala比较:

Impala性能稍领先于Presto，但是Presto在数据源支持上非常丰富，包括Hive、图数据库、传统关系型数据库、Redis等

# 优化

# 数据存储

指的是hive表

  • 合理设置分区
  • 使用列式存储：建议采用ORC
  • 使用压缩：建议采用Snappy压缩

# 查询SQL

  • 只选择使用的字段。

  • 能用分区做过滤条件，先用分区做过滤条件

  • Group BY 语句优化：按照积数降序排列

    [GOOD]: SELECT GROUP BY uid, gender
[BAD]:  SELECT GROUP BY gender, uid
    1
    2

  • Order by时使用Limit

  • 使用Join语句时将大表放在左边

# 注意事项

  • 字段名的引用：

    MySQL对字段加反引号`、Presto对字段加双引号分割
#如果不是关键字，就可以不加
    1
    2

  • 时间函数

    对于Timestamp，需要进行比较的时候，需要添加Timestamp关键字，而MySQL中对Timestamp可以直接进行比较。
/*MySQL的写法*/
SELECT t FROM a WHERE t > '2017-01-01 00:00:00'; 

/*Presto中的写法*/
SELECT t FROM a WHERE t > timestamp '2017-01-01 00:00:00';
