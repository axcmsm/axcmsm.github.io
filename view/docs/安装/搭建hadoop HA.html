# 搭建hadoop HA

  1. 前置知识以及搭建 hadoop zookeeper
  2. 规划节点 2个还是3个namenode 或更多 （2.6.0版本只能规划俩台节点，否则namenode报传参异常)

核心问题：如何保证数据一致性

  • namenode 数据一致

    - Fsimage 让一台nn生成数据,其他机器的nn同步。  文件
- Edits：需要引进新的模块 JournalNode 来保证edtis文件的数据一致性。  文件
    1
    2

  • 只有一台工作(active)其他待备份(standby)

    - 手动分配
- 自动分配 *
    1
    2

  • 2nn 在ha架构中并不存在

    定期合并fsimage,edtis的活由standby的nn来干
    1

  • nn'发生了问题如何切换上位

    - 手动故障转移
- 自动故障转移  zookeeper
    1
    2

	dfs.namenode.http-address.mycluster.nn1 => hadoop102:9870
7.2. 指定namenode的http通信地址nn2
	dfs.namenode.http-address.mycluster.nn2 => hadoop103:9870
7.3. 指定namenode的http通信地址nn3
	dfs.namenode.http-address.mycluster.nn3 => hadoop104:9870

	dfs.namenode.shared.edits.dir => qjournal://hadoop102:8485;hadoop103:8485;hadoop104:8485/mycluster

9. 访问代理类 => client 用于确定那个namenode 为Active
	dfs.client.failover.proxy.provider.mycluster => org.apache.hadoop.hdfs.server.namenode.ha.ConfiguredFailoverProxyProvider

	dfs.ha.fencing.methods => sshfence

11. 使用隔离级别需要ssh密钥登录  => 自动故障转移中使用到

# 自动模式 ⭕

最核心的解决上个模式遗留下来的最后一个问题.

添加的新组件:

  • Zookeeper(存储少量数据)和ZKFailoverController (ZKFC)(监控)

故障转移机制

  1. active出现假死状态
  2. ZKFC 检测到该状态 通知另一台NameNode 的ZKFC
  3. 另外一台的namde强行杀死假死的active防止脑裂 (ssh kil l- 9 进程号) 补刀失败 再调用用户自定义脚本再次补刀..(proweroff.sh),获取命令结果
  4. 激活namenode 为Active

集群规划: 3台Namenode,journalNode,datanode,zookeeper,ZKFC

配置文件:

  • hdfs-site.xml 添加

    1. 启动nn 故障自动转移
	dfs.ha.automatic-failover.enabled => true
    1
    2

  • core-site.xml 添加

    1. 指定zkfc 连接到zkService 地址
	ha.zookeeper.quorum => hadoop102:2181,hadoop103:2181,hadoop104:2181
    1
    2

# Yarn高可用

工作机制:

1.启动多个ResourceManager 最先启动的节点会到zookeeper中注册临时节点 active  (临时节点 进程挂掉节点自动删除)
2. 后启动页会尝试该操作(如果已由节点注册信息) standby
3. 所有的standby都需要维护一个长轮询,查看节点信息是否存在,一旦节点信息不存在,就立即注册.提升为Active
1
2
3

规划:

3台ResourceManager,NodeManager,zookeeper

核心问题:

  • active rm挂了 其他standby rm 如何上位 =>利用zk的临时节点
  • standby如何接手程序=> rm会将所有计算程序状态存储到zk中,其他rm上位后会去读取,接着跑

配置文件:

  • yarn-site.xml

    1. 读取数据的方式 shuffle
	yarn.nodemanager.aux-services => mapreduce_shuffle
2.  启用resourceManager ha
	yarn.resourcemanager.ha.enabled => true
3. 声明俩台resourcemanager 地址 集群名称
	yarn.resourcemanager.cluster-id => cluster-yarn1
4. 指定resourcemanager 逻辑列表
	yarn.resourcemanager.ha.rm-ids => rm1,rm2,rm3

5.配置具体地址
	yarn.resourcemanager.hostname.rm1 => hadoop102
	yarn.resourcemanager.hostname.rm2 => hadoop103
	yarn.resourcemanager.hostname.rm3 => hadoop104
6. 配置web端地址 rm1
	yarn.resourcemanager.webapp.address.rm1 => hadoop102: 8088
	yarn.resourcemanager.webapp.address.rm2 => hadoop103: 8088
	yarn.resourcemanager.webapp.address.rm3 => hadoop104: 8088
7.配置内部通信地址 rm1
	yarn.resourcemanager.address.rm1=hadoop102:8032
	yarn.resourcemanager.address.rm2=hadoop103:8032
	yarn.resourcemanager.address.rm3=hadoop104:8032
8. 配置Am向rm1申请支援的地址
	yarn.resourcemanager.scheduler.address.rm1 => hadoop102:8030
	yarn.resourcemanager.scheduler.address.rm2 => hadoop103:8030
	yarn.resourcemanager.scheduler.address.rm3 => hadoop104:8030
9. 提供NM联系的地址
	yarn.resourcemanager.resource-tracker.address.rm1 => hadoop102:8031
	yarn.resourcemanager.resource-tracker.address.rm2 => hadoop103:8031
	yarn.resourcemanager.resource-tracker.address.rm3 => hadoop104:8031
10. 指定zk
	yarn.resourcemanager.zk-address => hadoop102:2181,hadoop103:2181,hadoop104:2181

11. 启动自动恢复
	yarn.resourcemanager.recovery.enabled => true

12. resourcemanager 状态信息存储到zk集群
	yarn.resourcemanager.store.class => org.apache.hadoop.yarn.server.resourcemanager.recovery.ZKRMStateStore
	
13. 环境变量继承
	yarn.nodemanager.env-whitelist => JAVA_HOME,HADOOP_COMMON_HOME,HADOOP_HDFS_HOME,HADOOP_CONF_DIR,CLASSPATH_PREPEND_DISTCACHE,HADOOP_YARN_HOME,HADOOP_MAPRED_HOME

    • 分发脚本：vim /home/axcmsm/bin/xsync

      #!/bin/bash
#1. 判断参数个数
if [ $# -lt 1 ]
then
  echo Not Enough Arguement!
  exit;
fi
#2. 遍历集群所有机器
for host in master slave1 slave2
do
  echo ====================  $host  ====================
  #3. 遍历所有目录，挨个发送
  for file in $@
  do
    #4 判断文件是否存在
    if [ -e $file ]
    then
      #5. 获取父目录
      pdir=$(cd -P $(dirname $file); pwd)
      #6. 获取当前文件的名称
      fname=$(basename $file)
      ssh $host "mkdir -p $pdir"
      rsync -av $pdir/$fname $host:$pdir
    else
      echo $file does not exists!
    fi
  done
 done
